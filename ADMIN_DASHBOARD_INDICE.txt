📋 ENDPOINTS
───────────────────────────────────────────────────────────────────────────────

POST /api/admin/register
├─ Body: { email, password, company_name, full_name }
├─ Response: { success, token, user }
└─ Status: 200 ou 400 (duplicate email)

POST /api/admin/login
├─ Body: { email, password }
├─ Response: { success, token, user }
└─ Status: 200 ou 401 (invalid credentials)

POST /api/admin/tokens?token=JWT
├─ Body: { whatsapp_token, telegram_token, ... }
├─ Response: { success, message, tokens_count }
└─ Status: 200 ou 401 (invalid token)

GET /api/admin/tokens?token=JWT
├─ Response: { tokens (masked), configured, total_channels }
└─ Status: 200 ou 401

DELETE /api/admin/tokens/{channel}?token=JWT
├─ Channel: whatsapp, telegram, email, meta
├─ Response: { success, message }
└─ Status: 200 ou 404

GET /api/admin/status?token=JWT
├─ Response: { user, channels {status}, configured_count, apis_status }
└─ Status: 200 ou 401

GET /api/admin/user?token=JWT
├─ Response: { email, company_name, full_name, created_at, tokens_configured }
└─ Status: 200 ou 401

GET /login
├─ Response: HTML página de login/registro
└─ Status: 200

GET /dashboard
├─ Response: HTML painel de controle
├─ Note: Requer token JWT válido em localStorage
└─ Status: 200

✨ EXEMPLOS DE USO
───────────────────────────────────────────────────────────────────────────────

1. REGISTRAR NOVO CLIENTE
$ curl -X POST http://localhost:8000/api/admin/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "cliente@empresa.com",
    "password": "Senha123!",
    "company_name": "Minha Empresa",
    "full_name": "Cliente Nome"
  }'

Response:
{
  "success": true,
  "message": "Usuário registrado com sucesso!",
  "token": "eyJhbGc...",
  "user": {
    "email": "cliente@empresa.com",
    "company_name": "Minha Empresa",
    "full_name": "Cliente Nome"
  }
}

2. FAZER LOGIN
$ curl -X POST http://localhost:8000/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "cliente@empresa.com",
    "password": "Senha123!"
  }'

Response:
{
  "success": true,
  "token": "eyJhbGc...",
  "user": { ... }
}

3. ADICIONAR TOKENS
$ curl -X POST 'http://localhost:8000/api/admin/tokens?token=eyJhbGc...' \
  -H "Content-Type: application/json" \
  -d '{
    "whatsapp_token": "EAA1234567890",
    "whatsapp_phone_id": "123456",
    "telegram_token": "123456:ABC-DEF",
    "sendgrid_key": "SG.sk_live_xxxx",
    "email_from": "noreply@empresa.com"
  }'

Response:
{
  "success": true,
  "message": "Tokens salvos com sucesso!",
  "tokens_count": 5
}

4. VER TOKENS (MASCARADOS)
$ curl http://localhost:8000/api/admin/tokens?token=eyJhbGc...

Response:
{
  "tokens": {
    "WHATSAPP_ACCESS_TOKEN": "EAA...***...890",
    "WHATSAPP_PHONE_ID": "123...***...456",
    ...
  },
  "configured": 5,
  "total_channels": 4
}

5. VER STATUS
$ curl http://localhost:8000/api/admin/status?token=eyJhbGc...

Response:
{
  "user": {
    "email": "cliente@empresa.com",
    "company": "Minha Empresa",
    "name": "Cliente Nome",
    "created_at": "2024-01-15T10:30:00"
  },
  "channels": {
    "whatsapp": true,

    "email": true,
