🔐 SEGURANÇA - DETALHES TÉCNICOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

JWT AUTHENTICATION:
  ├─ Algorithm: HS256
  ├─ Expiration: 24 horas (configurável)
  ├─ Secret Key: Via environment variable (RAILWAY)
  └─ Payload: { "sub": email, "exp": timestamp }

PASSWORD HASHING:
  ├─ Algorithm: bcrypt
  ├─ Salt rounds: 12 (industry standard)
  ├─ Impossible to reverse: ✓
  └─ Constant time comparison: ✓

TOKEN MASKING:
  ├─ Request: "EAA1234567890abcdefghij"
  ├─ Response: "EAA...***...hij"
  ├─ Only last 5 chars visible: ✓
  └─ Prevents shoulder surfing: ✓

EMAIL VALIDATION:
  ├─ Using: Pydantic EmailStr
  ├─ RFC 5322 compliant: ✓
  └─ Prevents invalid emails: ✓

CORS SECURITY:
  ├─ Origins: Configuráveis
  ├─ Credentials: Allowed
  ├─ Methods: GET, POST, DELETE, OPTIONS
  └─ Headers: Content-Type, Authorization

ERROR HANDLING:
  ├─ 400: Bad request (validação)
  ├─ 401: Unauthorized (token inválido)
  ├─ 404: Not found
  └─ 500: Server error (sem stack trace em prod)
